# Infra_04_Rewrite_ci_for_nextjs

## Goal
Right now CI just runs `ls -la` — a placeholder. We're upgrading it to actually catch bugs: install dependencies, run TypeScript, run ESLint, run a Next.js production build. From this commit on, every push and PR is automatically checked. We also generate the `pnpm-lock.yaml` and fix two Next.js 14 config quirks discovered while replaying the build locally.

## Files changed
- `plans/Infra_04_Rewrite_ci_for_nextjs.md` — **created**: this file.
- `.github/workflows/ci.yml` — **edited**: replaced the placeholder job with a real `build` job that installs via pnpm, then runs typecheck → lint → build. Also: triggers on **all branches** (not just `main`), and adds `concurrency` so old runs cancel when you push new commits.
- `pnpm-lock.yaml` — **created**: pins exact dependency versions. Generated by `pnpm install`. CI uses `--frozen-lockfile`, so this file MUST be committed for CI to pass.
- `next.config.ts` — **deleted**: Next.js 14 doesn't read TypeScript config files (that's a Next 15 feature). We swap to `.mjs`.
- `next.config.mjs` — **created**: same config, just plain ESM JavaScript with a JSDoc type hint.
- `eslint.config.mjs` — **deleted**: Next 14's `next lint` doesn't yet understand the new flat config — it kept prompting interactively to set up ESLint.
- `.eslintrc.json` — **created**: legacy ESLint config that `next lint` happily reads. Same rules: `next/core-web-vitals` + `next/typescript`.

## Verification
1. Push the branch to GitHub: `git push`.
2. Open the repo's **Actions** tab — the `CI` workflow should show a `build` run starting.
3. After ~1–2 minutes, the run should be green ✅. Click in to confirm: install → typecheck → lint → build all passed.
4. (Optional) Try breaking it: `pnpm typecheck` will fail if you introduce a TS error locally — the same failure shows up in CI on next push.

## Gotchas / decisions
- **Two Next.js 14 quirks fixed mid-flight.** I originally added `next.config.ts` and `eslint.config.mjs` in Phase 2 — both unsupported by Next 14 (they're Next 15 features). Build and lint failed locally on first replay. Swapped to `next.config.mjs` and legacy `.eslintrc.json`. Lesson: always run a local CI replay before pushing.
- **`pnpm-lock.yaml` is committed.** CI uses `pnpm install --frozen-lockfile` for reproducible installs — that flag requires the lockfile to exist in the repo. `node_modules/` stays gitignored.
- **Triggers on all branches now**, not just `main`. That way feature branches like `day01` get checked before you open a PR. Keeps you fast.
- **`concurrency` block** — if you push twice in 30 seconds, the first run gets cancelled. Saves CI minutes and you only see the result for your latest commit.
- **Prisma generate is skipped** — it would need `DATABASE_URL` as a secret, which we don't have set in the repo yet. We'll add a `prisma:generate` step on the day we wire up the database.
- **No tests yet** — there's nothing to test in a scaffold. We'll add a `test` step the day we add our first test (likely with Vitest).
- **pnpm cache** — the workflow caches the pnpm store between runs. First run is ~2 min; subsequent runs are usually under 1 min.
